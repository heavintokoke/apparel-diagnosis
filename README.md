# 服装企业经营管理提效工具

这是一个面向服装企业的经营管理提效总工作台。项目目标不是做单一工具，而是把诊断、战略、组织、商品、供应链、库存、门店、内容、运营、数据和 AI 提效逐步整合到同一个网页系统中。

## 当前阶段

第一阶段：总工作台框架 + 企业经营诊断系统正式接入。
第二阶段：公共能力底座 + 统一资料中心已完成基础版。

已完成：

- 16 大模块总入口
- 总驾驶舱页面
- 模块详情切换
- 建设路线页面

- 公共文件识别：Word、Excel、PDF、图片 OCR、CSV、文本统一抽取
- 公共 AI 提示词与报告中心索引

## 打开方式

当前已经包含后端能力，请用本地服务方式打开：

```bash
python3 -m venv .venv
. .venv/bin/activate
pip install -r requirements.txt
./run.sh
```

然后访问：

```text
http://127.0.0.1:8770
```

经营诊断模块：

```text
http://127.0.0.1:8770/modules/01-enterprise-diagnosis/
```

## 后续开发原则

- 所有新增功能都优先作为模块接入总工作台，不再单独做成孤立项目。
- 公共能力要沉淀为共享底座，例如资料中心、报告中心、指标体系、AI 提示词库。
- 每个模块都按统一结构建设：输入资料、分析逻辑、业务动作、复盘指标、导出报告。
- 静态页面可用于展示系统规划和阶段成果；正式业务功能通过后端模块接入。

## 公共能力与资料中心

- `/api/materials`：统一上传、识别、分类和列出企业资料。
- `/api/analyze`：经营诊断模块现在从资料中心读取资料包；上传新资料时会先进入资料中心，再被诊断模块引用。
- `/api/reports`：统一记录模块报告和导出文件。
- `app/extractors.py`、`app/ai_client.py`、`app/reporting.py`、`app/data_center.py` 是后续模块共用的底座，不再为每个模块重复建设上传、识别、提示词和导出逻辑。
